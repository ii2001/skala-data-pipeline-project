# 최종 모델 선택 기록

## 결정

여섯 팀원의 접근을 검토한 뒤 최종 모델로 **Lee Hyeonjun의 LightGBM 직접 회귀**를
선정했습니다.

## 검토한 팀 실험

| 작성자 | 접근 | 기존 평가 조건 | 검토 시 주의사항 |
|---|---|---|---|
| Kim Yechan | 경로 통계 + Ridge | 시간 홀드아웃, 50만 행 표본 | 공통 랜덤 분할과 다름 |
| Jeong Dayun | 9개 요금 다중출력 Random Forest | 20만 행 표본 | 기존 결과는 공통 평가 전 기록 |
| Lee Hyeonjun | `total_amount` LightGBM 직접 회귀 | 초기 랜덤 분할 | 최종 코드에서 공통 분할로 재구현 |
| Kim Heeju | 요금 항목별 LightGBM | 5월 1~24일/25~31일 시간 분할 | 직접 회귀와 타깃 구조가 다름 |
| Noh Yeongo | 승차 시점 Linear Regression | 공통 80:20 분할 | 해석 가능한 pickup-time 기준선 |
| Shin Minseo | 9개 요금 다중출력 Random Forest | 5-fold 교차검증 | 직접 회귀와 평가 계약이 다름 |

실험별 표본, 분할, 타깃 구조와 예측 시점이 달라 과거 MAE·R²를 하나의 순위표로
섞지 않았습니다. 특히 승차 시점 모델과 운행 종료 시점 모델은 사용 가능한 정보가
다르므로 성능 숫자만으로 우열을 판단할 수 없습니다.

## 선정 기준

1. 직접 요금 구성 9개 컬럼을 입력으로 사용하지 않을 것
2. 전처리와 모델을 하나의 sklearn Pipeline으로 저장할 것
3. 시간·거리·위치의 비선형 관계와 상호작용을 처리할 것
4. 전체 3,260,221개 학습 행을 사용할 수 있을 것
5. 테스트 815,056행을 제외 없이 예측할 것
6. 모델 저장 후 재로딩·재예측이 가능할 것
7. 팀이 합의한 운행 종료 직후 예측 시점과 일치할 것

## 공통 조건 최종 결과

|---|---:|

| MAE | 3.4002 |

| R² | 0.8754 |

## 최종 판단

LightGBM은 대규모 데이터와 비선형 관계를 효율적으로 처리했고, 전체 Pipeline 저장과
재로딩 검증까지 통과했습니다. 단, 60~100달러 구간 R²가 -0.2851이고 100달러 이상
MAE가 18.3387이므로 전체 성능만으로 모든 금액대가 정확하다고 해석하지 않습니다.

실행 근거는 `src/final/train.py`, 실제 수치는 `reports/final/metrics.json`, 전체
해석은 `reports/final/report.md`를 기준으로 합니다.

# 최종 모델 선택 기록

## 결정

팀 최종 모델로 **Lee Hyeonjun의 LightGBM 회귀 접근**을 선정했습니다.

## 비교 시 주의사항

팀원별 기존 결과는 평가 조건이 같지 않았습니다.

- Kim Yechan 실험: 시간 홀드아웃과 50만 행 표본 기반 Ridge
- Jeong Dayun 기존 실험: 20만 행 표본과 요금 구성요소 다중 출력 Random Forest
- Lee Hyeonjun 실험: LightGBM 기반 `total_amount` 직접 회귀

따라서 기존 MAE나 R² 숫자를 그대로 나열해 우열을 주장하지 않았습니다. 최종
선정 모델만 공통 원본, 공통 시드, 공통 80:20 분할과 전체 테스트 행으로 다시
검증했습니다.

## 선정 기준

1. 직접 요금 구성 9개 컬럼을 입력으로 사용하지 않을 것
2. 전처리와 모델을 하나의 sklearn Pipeline으로 저장할 것
3. 시간·거리·위치의 비선형 관계와 상호작용을 처리할 것
4. 전체 3,260,221개 학습 행을 사용할 수 있을 것
5. 테스트 815,056행을 제외 없이 예측할 것
6. 모델 저장 후 재로딩·재예측이 가능할 것

## 공통 조건 최종 결과

|-----------|----------:|

| MAE       |    3.4002 |

| Median AE |    1.5451 |

## 최종 판단

LightGBM은 대규모 데이터와 비선형 피처 관계를 효율적으로 처리했고, 전체 Pipeline
저장과 재로딩 검증까지 통과했습니다. 다만 60~100달러 구간의 R²가 -0.2851이고
100달러 이상 MAE가 18.3387이므로, 최종 보고서에는 전체 성능뿐 아니라 고액
운행의 한계도 함께 제시했습니다.

실행 근거는 `src/final/train.py`, 실제 수치는 `reports/final/metrics.json`, 전체
해석은 `reports/final/report.md`를 기준으로 합니다.

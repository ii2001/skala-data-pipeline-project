# 5분 발표 대본

## 0:00~0:35 — 문제와 데이터 선택

저희 팀은 NYC TLC Yellow Taxi 2026년 5월 데이터 409만 건을 선택했습니다.
시각·거리·위치·결제·금액처럼 다양한 타입과 실제 결측이 있어 Pandas·Polars 비교,
EDA, 통계 검정, 회귀 Pipeline을 한 데이터로 수행할 수 있기 때문입니다. 목표는
운행 종료 직후 `total_amount`를 예측하는 것입니다.

## 0:35~1:20 — 데이터 품질과 전처리

Pandas와 Polars가 4,090,836행 × 20열, 컬럼과 결측 건수에서 일치함을 확인했습니다.
중복은 0건이었습니다. `passenger_count`, `RatecodeID`, `store_and_fwd_flag`에는
각각 약 95만 건의 결측이 있어 Pipeline 내부에서 0, 99, Unknown으로 처리했습니다.
`total_amount`를 직접 구성하는 요금 9개 컬럼은 누수를 막기 위해 입력에서
제외했습니다. 거리와 duration은 이상값을 제한하고 보정 여부를 피처로 남겼습니다.

## 1:20~2:05 — EDA와 시각화

거리 분포는 오른쪽 꼬리가 길고, 거리와 총액의 Pearson 상관은 0.7845였습니다.
duration과 총액의 상관은 0.6632였고 승객 수와 총액의 상관은 0.0034로 거의
없었습니다. Seaborn으로 거리 분포와 상관 히트맵을 만들고, Plotly로 시간대별 평균·
중앙값과 운행량을 인터랙티브하게 비교했습니다.

## 2:05~2:45 — 통계 검정

신용카드와 현금 결제의 평균 총액을 Welch t-test로 비교했습니다. 신용카드 평균은
30.66달러, 현금은 26.11달러였고 t=108.71, p-value는 0.05보다 작았습니다.
귀무가설은 기각하지만 Cohen's d가 0.1948이므로, 큰 표본에서 통계적 유의성과
실질적 차이를 구분해 해석했습니다.

## 2:45~3:50 — 모델 선택과 Pipeline

여섯 팀원의 Ridge, Linear Regression, Random Forest, LightGBM 접근을 검토했습니다.
기존 실험은 분할과 타깃 구조가 달라 과거 점수를 그대로 순위화하지 않았습니다.
최종적으로 비선형 관계와 대규모 희소 데이터를 효율적으로 처리하는 Lee Hyeonjun의
LightGBM 직접 회귀를 선택했습니다. 숫자 결측 처리와 범주형 원핫 인코딩,
LightGBM을 하나의 sklearn Pipeline으로 묶어 joblib으로 저장했습니다.

## 3:50~4:30 — 평가 결과

공통 80:20 분할로 학습 3,260,221행, 테스트 815,056행 전체를 예측했습니다.
결과는 MAE 3.4002달러, RMSE 7.7800달러, Median AE 1.5451달러, R² 0.8754입니다.
평균만 예측하는 기준선 MAE 14.5230보다 76.59% 감소했습니다. 연속형 회귀
문제이므로 정확도·F1 대신 이 회귀 지표들을 사용했습니다. 저장 모델을 다시 불러
같은 예측이 나오는 것도 검증했습니다.

## 4:30~5:00 — 한계와 개선

60~100달러 구간 R²는 음수이고 100달러 이상 MAE는 18.34달러로 고액 운행에서
오차가 커졌습니다. 또한 duration과 결제수단을 사용하므로 승차 전 가격 예측이 아닌
운행 종료 직후 모델입니다. 후속 과제로 시간 순서 외부 검증, 희귀 위치 통합,
고액 구간 가중치와 LightGBM 파라미터 검증을 제안합니다.
